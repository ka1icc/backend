# Что делать по шагам

---

## Загрузить задание Avito заново (как 2-е задание) + закрыть старый PR

### Шаг 0. Закрыть старый PR по заданию 1

1. Зайди на https://github.com/ka1icc/backend
2. Вкладка **Pull requests**
3. Открой PR из ветки **1-avito** в main
4. Внизу страницы нажми **Close pull request**
5. Старый PR будет закрыт, ветку 1-avito не удаляй

### Шаг A. Взять за основу main и пересобрать ветку 1-avito

В **Git Bash**:

```bash
cd ~/Desktop/backend-repo
git fetch origin
git checkout main
git pull origin main
git branch -D 1-avito
git checkout -b 1-avito
```

### Шаг B. Положить в репозиторий только папку с заданием 1

Скопируй **всё содержимое** папки `C:\Users\yulia\Desktop\1_backend_avito-tech` в папку **backend-repo/1_backend_avito-tech** (папку `1_backend_avito-tech` создай внутри backend-repo, если её нет). Не копируй лишнее (например `__pycache__`, `shortener.db`, `.idea` — можно не копировать).

Затем:

```bash
git add 1_backend_avito-tech
git status
git commit -m "Задание 1: Avito-tech URL Shortener"
git push -u origin 1-avito --force
```

`--force` перезапишет ветку 1-avito на GitHub.

### Шаг D. Создать новый PR

1. На GitHub: **Compare & pull request** для ветки **1-avito** (или **Pull requests** → **New pull request**: base **main**, compare **1-avito**)
2. Название: «Задание 1: Avito-tech URL Shortener»
3. **Create pull request** — в изменениях будут только добавления (как у задания 2)

## 1. Почистить ветку 1-avito (оставить только задание Avito)

*(Если ветка 1-avito уже есть и в ней лишнее — только убрать Ivelum.)*

В **Git Bash** в папке **backend-repo**:

```bash
cd ~/Desktop/backend-repo
git checkout 1-avito
git pull origin 1-avito
git rm -r 2_backend_Ivelum
git commit -m "1-avito: только задание Avito"
git push origin 1-avito
```

В ветке **1-avito** останутся только README и папка **1_backend_avito-tech**.

---

## 2. Сделать Pull Request для преподавателя

1. Зайди на https://github.com/ka1icc/backend
2. Выбери ветку **1-avito**
3. Нажми **Compare & pull request** (или **Pull requests** → **New pull request**)
4. Base: **main**, Compare: **1-avito**
5. Напиши название, например: «Задание 1: Avito-tech URL Shortener»
6. Нажми **Create pull request**
7. Отправь ссылку на этот PR преподавателю — он будет оставлять комментарии к коду там

**PR не мержить** — пусть остаётся открытым для проверки. Тогда в main ничего не изменится и ничего не удалится.

## 3. Задание 2 (Ivelum) — отправить на ветку и сделать PR

### 3.1. Перейти на ветку 2-ivelum и положить туда код

В **Git Bash**:

```bash
cd ~/Desktop/backend-repo
git checkout 2-ivelum
git pull origin 2-ivelum
```

Скопируй все файлы второго задания (Ivelum) в папку **backend-repo/2_backend_Ivelum**.  
Если папки `2_backend_Ivelum` нет — создай её внутри backend-repo и скопируй туда файлы проекта (как у тебя лежит задание 2 на диске).

Затем в Git Bash:

```bash
git add 2_backend_Ivelum
git status
git commit -m "Задание 2: Ivelum"
git push -u origin 2-ivelum
```

### 3.2. Создать Pull Request для задания 2

1. Зайди на https://github.com/ka1icc/backend
2. Переключись на ветку **2-ivelum** (выпадающее меню веток)
3. Нажми **Compare & pull request** (или **Pull requests** → **New pull request**)
4. Base: **main**, Compare: **2-ivelum**
5. Название, например: «Задание 2: Ivelum»
6. **Create pull request**
7. Отправь ссылку на этот PR преподавателю

**PR не мержить** — только для проверки и комментариев.
